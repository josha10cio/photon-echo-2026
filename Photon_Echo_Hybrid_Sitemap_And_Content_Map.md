# Photon Echo Hybrid Sitemap and Content Map

## Goal

Create a hybrid version of `photonecho.dev` that preserves the strongest structural work already present on the live site while replacing the messaging with the updated, broader, customer-facing positioning developed in this project.

The new site should:

- Keep the useful multi-page structure already built into the current site
- Preserve valuable navigation, page inventory, and content effort from the live version
- Replace investor-first framing with customer-facing positioning
- Use broader language for companies building and deploying real-world AI systems
- Avoid vendor-centered messaging

## What Exists on the Current Site

### Homepage

Current homepage themes:

- Simulation infrastructure
- Structured synthetic data
- Robotics perception
- Customer contact CTA

Current issues:

- Still carries investor-oriented navigation and CTA structure
- Reads more like an early-stage concept site than a full customer-facing company site
- Some useful sections exist, but hierarchy and flow need to be reorganized

### Platform Build (`/phase-i`)

What it does now:

- Explains the first demonstration phase of the simulation workflow
- Describes environment construction, behavior modeling, sensor simulation, scenario variation, labeling, and validation

Assessment:

- Useful content
- Current name is too internal and phase-based
- Better reframed as a product/process page

Recommendation:

- Keep the content direction
- Rename and rewrite into `How It Works` or `Platform`

### Market (`/market`)

What it does now:

- Explains where Photon Echo fits in the market
- Talks about the transition from experimental robotics to deployed autonomy
- Identifies target segments

Assessment:

- Useful strategic positioning
- Good source material for customer-facing market and use-case language
- Current page can be merged into a broader `Industries` or `Who It Serves` page

Recommendation:

- Merge core ideas into homepage industries section
- Optionally keep a standalone `Industries` page if you want a fuller site

### Investors (`/investors`)

What it does now:

- Investor-oriented page and CTA path

Assessment:

- Misaligned with the new direction for the public-facing site

Recommendation:

- Remove from primary navigation
- Keep only if you want a private or secondary investor page later

### Enterprise Pilot (`/early-access`)

What it does now:

- Explains the pilot structure
- Defines who it is for
- Encourages inquiry

Assessment:

- Strong page for customer conversion
- This is one of the most useful existing pages

Recommendation:

- Keep
- Rename in nav to `Pilot Program`, `Start a Pilot`, or `Enterprise Pilot`
- Rewrite using the new broader messaging

### FAQ (`/faq`)

What it does now:

- Covers the company, focus, data, customers, differences, and use cases

Assessment:

- Valuable supporting content

Recommendation:

### Mission (`/mission`)

What it does now:

- Explains mission, why the company exists, how it works, and focus areas

Assessment:

- Good source material

Recommendation:

### Government (`/government`)

## Recommended Messaging Rules for the Hybrid Site

- Lead with simulation, synthetic data, validation, and deployment confidence
- Speak to companies and programs, not just narrow technical builder groups
- Avoid investor-first framing on the public site
- Avoid centering the brand around specific software vendors
- Keep technical credibility, but focus on customer outcomes first
- Let use cases and industry examples explain the workflow instead of relying on a dedicated `How It Works` page
